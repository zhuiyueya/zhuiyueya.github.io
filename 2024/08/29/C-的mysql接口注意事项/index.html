C++的mysql接口注意事项

##多进程：

1.在父进程中初始化并连接的MYSQL，若在子进程也需要使用其命令，则需要在子进程中再次初始化并连接到mysql服务器
原因：虽然子进程获得了连接的副本，但每个 MySQL 连接都是独立的，子进程中的连接副本实际上是无效的，它不再与 MySQL 服务器保持活动的通信。

##接口的使用：

1.在执行一次mysql_query后，若未处理结果，如调用 mysql_store_result() 或 mysql_use_result()，就再次调用mysql_query,那么第二次调用将发生错误
